Необходимо создать пример работы динамических связанных выпадающих списков, создающихся на основе информации из базы данных MySQL.
Система должна иметь инсталлятор, который при запуске создает таблицы и данные из текстового sql-файла. Доступы к базе данных должны храниться в конфигурационном файле.
Структура базы данных
В базе данных должны быть следующие таблицы и поля:
страны (id, название);
города (id, id страны, название);
отели (id, id города, название).
Интерфейс
Готовое решение должно состоять из двух «экранов»:
экран инсталляции;
экран списков.
Экран инсталляции
При загрузке происходит проверка соединения с базой данных и проверка наличия текстового sql-файла. В случае какой-либо ошибки должно выводиться соответствующее сообщение. В случае, если ошибки отсутствуют — на странице должна находиться кнопка «Установить», по нажатию на которую должен происходить запуск инсталлятора и переход на экран списков (или отображение списков на этой же странице).
Экран списков
На странице должны быть 3 выпадающих списка — страна, город, отель. Значение по умолчанию для всех списков — «Не выбран». На странице должна находиться кнопка «Загрузить», по нажатию на которую должна происходить загрузка данных в списки. В случае если значение в родительском списке (страна, город) «Не выбран» — в дочернем списке (город, отель) должны быть все варианты из базы данных. При выборе любого значения из любого списка — в родительских списках должны выбираться соответствующие элементы (по связям через id), а дочерние списки должны фильтроваться (также по связям через id).
